# Brainstorming - User Stories

## Story 1: TA creates profile
**User Story**  
As a TA,  
I want to create my applicant profile,  
So that I can provide my personal and academic information for job applications.

**Acceptance Criteria**
- The system allows TA to enter personal information.
- The system allows TA to enter academic background.
- The profile can be saved successfully.
- The saved profile can be edited later.

**Priority**  
High

**Estimation**  
3 story points

**Iteration Plan**  
Sprint 1


## Story 2: TA uploads CV
**User Story**  
As a TA,  
I want to upload my CV,  
So that I can apply for teaching assistant positions.

**Acceptance Criteria**
- The system allows TA to upload a CV file.
- The system only accepts PDF format.
- The uploaded CV can be saved in the system.
- MO can view the uploaded CV.

**Priority**  
High

**Estimation**  
3 story points

**Iteration Plan**  
Sprint 1


## Story 3: TA views available jobs
**User Story**  
As a TA,  
I want to view available TA jobs,  
So that I can find suitable opportunities.

**Acceptance Criteria**
- The system displays a list of available jobs.
- Each job includes module name, description, and requirements.
- TA can click on a job to see more details.

**Priority**  
High

**Estimation**  
2 story points

**Iteration Plan**  
Sprint 1


## Story 4: TA applies for a job
**User Story**  
As a TA,  
I want to apply for a job,  
So that I can be considered for the position.

**Acceptance Criteria**
- TA can click an apply button for an available job.
- The system records the application successfully.
- TA can only apply after profile and CV are completed.
- A confirmation message is shown after submission.

**Priority**  
High

**Estimation**  
5 story points

**Iteration Plan**  
Sprint 1


## Story 5: TA checks application status
**User Story**  
As a TA,  
I want to check my application status,  
So that I can know whether I have been selected or not.

**Acceptance Criteria**
- TA can view all submitted applications.
- Each application shows its current status.
- Status can be displayed as pending, accepted, or rejected.

**Priority**  
Medium

**Estimation**  
3 story points

**Iteration Plan**  
Sprint 2


## Story 6: MO logs into the system
**User Story**  
As a MO,  
I want to log into the system securely,  
So that I can manage job postings and applicants.

**Acceptance Criteria**
- MO can enter username and password.
- The system verifies login credentials.
- Invalid login shows an error message.
- Successful login redirects MO to the MO dashboard.

**Priority**  
High

**Estimation**  
3 story points

**Iteration Plan**  
Sprint 1


## Story 7: MO posts a job
**User Story**  
As a MO,  
I want to post a TA job,  
So that students can apply for it.

**Acceptance Criteria**
- MO can create a new job post.
- The job post includes module name, job description, and requirements.
- The job becomes visible in the available jobs list after saving.

**Priority**  
High

**Estimation**  
5 story points

**Iteration Plan**  
Sprint 1

As a MO,

**Acceptance Criteria**
- MO can see the list of applicants for each posted job.
- Each applicant record shows name, profile, and CV.
- MO can open an applicant’s details page.

High

**Estimation**  
3 story points

Sprint 2

**Acceptance Criteria**

- The application status is updated after selection.

High

Sprint 2


## Story 10: Admin checks TA workload
**User Story**  
As an admin,  
I want to check each TA’s overall workload,  
So that I can avoid assigning too much work to one student.

**Acceptance Criteria**
- Admin can view all assigned TA jobs.
- The system shows workload for each TA.
- Admin can identify overloaded TAs easily.

**Priority**  
Medium

**Estimation**  
5 story points

**Iteration Plan**  
Sprint 3

**Acceptance Criteria**

- Admin can search for an applicant by name.

Medium

Sprint 3


## Story 12: TA receives feedback on missing skills
**User Story**  
As a TA,  
I want to know what skills are missing for a job,  
So that I can improve my future applications.

**Acceptance Criteria**

- The system shows missing skills or qualifications.
- The feedback is clearly displayed to the TA.

**Priority**  
Low

**Estimation**  
5 story points

**Iteration Plan**  
Sprint 4


## Story 13: System matches jobs and applicants
**User Story**  
As a MO,  
I want the system to suggest suitable applicants for a job,  
So that I can make decisions more efficiently.

**Acceptance Criteria**
- The system compares applicant skills with job requirements.
- The system generates a suggested matching result.
- The matching result is visible to MO.

**Priority**  
Low

**Estimation**  
8 story points

**Iteration Plan**  
Sprint 4


## Story 14: Admin balances workload
**User Story**  
As an admin,  
I want the system to warn me when a TA has too much workload,  
So that I can distribute assignments fairly.

**Acceptance Criteria**
- The system calculates total workload for each TA.
- The system highlights TAs with heavy workload.
- Admin can view this warning before confirming assignments.

**Priority**  
Low

**Estimation**  
5 story points

**Iteration Plan**  
Sprint 4
